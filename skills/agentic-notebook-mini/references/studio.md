# Studio artifacts — mini templates

Shared rules (all artifacts): grounded in sources only · every substantive
claim cited [n] · one Sources footer at the end (same format as answers) ·
save to `<folder>/.notebook/studio/<name>-<date>.md` · run verify_citations.py
and fix FAILs · honor user knobs (focus, audience, length, language) · build
from targeted searches + `--get` reads, never from memory; for whole-corpus
artifacts run a Mode B ledger first and compose from its output file.

**Briefing doc** — # Title · Executive summary (1 para) · Key themes (3-5,
each 2-4 sentences + citations) · Notable facts & figures (cited list) ·
Conclusion (1 para).

**Study guide** — 8-10 short-answer questions + answer key (each answer
cited) · 3 essay prompts · glossary of 8-12 key terms (definitions from the
sources, cited).

**FAQ** — 6-10 questions a real reader would ask, each answered in 2-4
sentences with citations. Questions must be answerable from the sources.

**Timeline** — chronological `YYYY (or period) — event [n]` lines · then
"Cast of characters": each name + 1-2 sentence bio, cited.

**Flashcards** — ~20 cards `Q: ... / A: ... [n]` · offer a CSV
(`front,back`) for Anki alongside.

**Quiz** — 8-12 MCQs (A-D) · answer key at the end: correct letter +
1-sentence explanation + citation each.

**Mind map** — indented outline (central topic → 3-6 themes → subtopics,
leaf nodes cited) · plus a ```mermaid mindmap``` block mirroring it.

**Data table** — markdown table of facts scattered across sources; one
"source" column with [n] per row; offer CSV.

**Podcast script (audio overview)** — 2 hosts (A curious, B explains) ·
target ~700-900 words unless asked · structure: hook → 3-4 segments (one
theme each) → recap + one open question · spoken text stays clean; citations
go in a "Production notes" section at the end, mapping each segment to its
sources.

**Custom report** — whatever format the user names; same shared rules.
